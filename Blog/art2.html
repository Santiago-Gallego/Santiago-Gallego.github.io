Optimización de Consultas en SQL Server

Escrito por Santiago Gallego

¿Por qué optimizar consultas?

La optimización de consultas SQL es crucial porque permite ejecutar consultas de forma más eficiente, reduciendo el tiempo de ejecución y el uso de recursos, lo que mejora el rendimiento general de la base de datos y la aplicación.

Técnicas Clave

1. Índices Estratégicos

Crea índices en columnas usadas frecuentemente en WHERE, JOIN y ORDER BY:

CREATE INDEX idx_usuario_email ON Usuarios(email);

En lugar de leer todas las páginas (escaneo completo de tabla), te permiten saltar directamente a la información que buscas.

2. Evita SELECT * FROM "Tu_Tabla"

Selecciona solo las columnas necesarias:

-- Mal
SELECT * FROM Productos;

-- Bien
SELECT id, nombre, precio FROM Productos;

Esto para optimizar la respuesta ya que al traer menos datos tenemos menos tiempos de respuesta, eto tambien implica que el rendimiento la seguridad y la mantenibilidad es mejor.

Análisis de Rendimiento

Usa estas consultas para monitorear:

-- Consultas más lentas
SELECT TOP 10 
  query_stats.query_hash,
  query_stats.execution_count,
  query_stats.total_elapsed_time / query_stats.execution_count AS avg_duration
FROM sys.dm_exec_query_stats AS query_stats
ORDER BY avg_duration DESC;

Esta consulta es una herramienta poderosa para diagnóstico de rendimiento en SQL Server. Te ayuda a identificar las consultas más lentas en tu base de datos. Extrae las 10 consultas mas lentas (en promedio) ejecutadas en tu SQL Server basandose en:

  • query_hash:

    Identificador único del tipo de consulta (consultas idénticas tienen el mismo hash).

  • execution_count:

    Veces que se ejecutó esa consulta.

  • avg_duration:

    Tiempo promedio de ejecución (en milisegundos).
